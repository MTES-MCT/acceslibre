{% translate "1. Introduction" %}

{% translate "Ce document présente la politique d’accessibilité d'Acceslibre." %}

{% translate "Acceslibre est une plateforme collaborative, à la fois dédiée aux professionnels (gérants d'établissements, administration) et au grand public." %}

{% blocktranslate trimmed %} Acceslibre fait partie du réseau des Startup d'Etat coordonnées par la direction du numérique, DINUM, qui met à disposition des moyens et outils pour les produits numériques du réseau beta.gouv.fr, et qui a son propre Schéma Pluriannuel d'Accessibilité Numérique. {% endblocktranslate %}

{% translate "2. Ressources humaines et techniques" %}

{% translate "L'équipe est constituée de 6 personnes en 2026. Les personnes en charge de l'accessibilité numérique sont principalement les intrapreneuses du projet et les deux développeurs." %}

{% translate "Des experts de la DINUM peuvent être mobilisés pour conseiller mais également réaliser des audits RGAA pour les startups d'Etat." %}

{% translate "3. Politique d'accessibilité numérique" %}

{% translate "3.1 Objectifs généraux" %}

{% translate "Acceslibre se fixe les objectifs suivants en matière d’accessibilité numérique :" %}

  • {% translate "Sensibiliser et former à l’accessibilité les membres de l'équipe impliqués dans la création et la gestion de produits et de contenus numériques ;" %}
  • {% translate "Intégrer l’accessibilité numérique dès la conception des nouveaux services et fonctionnalités ;" %}
  • {% translate "Atteindre et maintenir un niveau de conformité élevé au RGAA - si possible 100% ;" %}
  • {% translate "Mettre en place des tests utilisateurs réguliers dans un objectif d'amélioration continue de l'accessibilité du site ;" %}
  • {% translate "Vérifier l'accessibilité numérique de sites partenaires et, pour les sites non conformes, les inciter à se rendre accessibles ;" %}
  • {% translate "Veiller à l'accessibilité des différentes communications : newsletter, scorecard, post LinkedIn..." %}

{% translate "3.2 Respect des obligations légales déclaratives" %}

{% translate "Tout développement réalisé sur le site :" %}

  • {% translate "respecte le DSFR ;" %}
  • {% translate "respecte le référentiel général d’amélioration de l’accessibilité (RGAA) dans sa version en vigueur." %}

{% translate "La déclaration d'accessibilité est vérifiée a minima une fois par an." %}

{% translate "3.3 Conception / Développement" %}

{% translate "Les principaux bénéficiaires du site Acceslibre sont les personnes en situation de handicap et leurs proches. Aussi, toutes les nouvelles fonctionnalités qui concernent l'usage sont conçues et testées au sein de groupe de travail avec les personnes concernées." %}

{% translate "L'équipe de développement sollicite l'équipe d'appui de la DINUM si nécessaire pour concevoir ou développer certaines fonctionnalités." %}

{% translate "Une clause concernant le respect du RGAA sera intégrée à toute mission externalisée." %}

{% translate "3.4 Contrôle" %}

{% blocktranslate trimmed %} Des tests automatiques ou automatisés sont mis en place. Un suivi est possible via le Dashlord. {% endblocktranslate %}

{% translate "Des tests manuels sont effectués lors de signalement ou de la mise en ligne de nouvelles fonctionnalités avec le logiciel VoiceOver." %}

{% translate "Il peut être fait appel à des personnes en situation de handicap pour tester certaines fonctionnalités." %}

{% translate "4. Actions réalisées en 2025" %}

{% translate "4.1 Refonte du site et audit" %}

{% translate "En 2025, les actions suivantes ont été mises en œuvre :" %}

  • {% translate "Refonte globale du site ayant pour objectif :" %}
    • {% translate "l'amélioration de l'ergonomie et du design ;" %}
    • {% translate "la mise en conformité totale au DSFR ;" %}
    • {% translate "la mise en accessibilité totale en vue d'un audit." %}

{% translate "Ce chantier sera essentiellement mené par une prestation externe d'UX/ design et le développeur de l'équipe plus spécifiquement en charge du front et de l'accessibilité." %}

{% translate "Audit RGAA par la DINUM." %}

{% translate "Compte tenu du plan de charge de la DINUM, l'audit a été reporté en 2026." %}

{% translate "4.2 Conception, développement, contrôle" %}

{% translate "Plusieurs ateliers utilisateurs ont été menés dans le cadre d'évolution du site notamment :" %}

  • {% translate "mise en place de filtres de recherche ;" %}
  • {% translate "évolutions du modèle de données." %}

{% translate "Partenariat avec la CFPSAA et l'association Valentin Haüy dans une démarche d'amélioration continue du site." %}

{% translate "5. Plan d'actions 2026" %}

{% translate "En 2026, les actions suivantes seront menées :" %}

  • {% translate "Audit (audit, corrections des non conformités, contre-audit) ;" %}
  • {% translate "Mise à jour de la déclaration d’accessibilité ;" %}
  • {% translate "Renforcement de l’approche “Accessibilité dès la conception” en associant à toutes évolutions du site les usagers concernés ;" %}
  • {% translate "Maintien du 100% accessible en veillant à ce que toutes les évolutions du site soient accessibles ;" %}
  • {% translate "Mise en place de contrôle de l'accessibilité numérique lors du recettage d'évolutions du produit ;" %}
  • {% translate "Maintien des compétences en matière d'accessibilité numérique des développeurs, en particulier, mise en place d'une formation spécifique en cas de changement de développeurs ;" %}
  • {% translate "Veiller à l'accessibilité des différentes communications : newsletter, scorecard, emailing, post LinkedIn... ;" %}
  • {% translate "Au-delà du site Acceslibre, communication auprès de nos partenaires sur l'importance de rendre accessible leur site." %}
